# 🔐 Security Policy

## 🧩 Supported Versions
The following table outlines which versions of this project currently receive security updates. Versions marked as supported will receive patches for confirmed security vulnerabilities.

| Version | Supported |
|--------|-----------|
| 5.1.x  | ✔️ Yes    |
| 5.0.x  | ❌ No     |
| 4.0.x  | ✔️ Yes    |
| < 4.0  | ❌ No     |

## 🛡️ Reporting a Vulnerability
If you discover a security vulnerability, we strongly encourage responsible disclosure. Please follow the process below:

### 📬 How to Report
- Submit your report via **GitHub Security Advisories** or email ([HIDDEN]).
- Include as much detail as possible:
  - Steps to reproduce  
  - Potential impact  
  - A proof-of-concept if available  
  - Your environment (OS, version, etc.)

### ⏱️ Response Expectations
- You will receive an acknowledgment within **48–72 hours**.
- We will provide status updates at least **once per week** until the issue is resolved.
- If the vulnerability is confirmed:
  - We will work on a fix promptly.
  - You may be asked for additional details or verification.
  - A coordinated disclosure timeline will be agreed upon.

### 🤝 What Happens Next
- If accepted, the issue will be patched in the next supported release.
- If declined, we will explain why (e.g., not reproducible, expected behavior, out of scope).
- Credit will be given to the reporter unless anonymity is requested.
